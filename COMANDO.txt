>Inciar una proyecto
git init

>Ver el estado de los archivos
git status 

>Agregar un archivo o un listado de rhviso al staging 
git add nombre_archivo

>Quitar el archivo del staging
git rm --cache nombre_archivo

Agregar el archivo al master o la rama indicada
>git commit  archivo_nuevo.tx -m 'Comentario'

Ver el historial de una archivo
>git log nombre_archivo.txt

Crear una rama
>git branch test

Acceder a una rama 
>git checkout test

Unir la informacion de dos ramas, se tiene que utilizar en la rama que quiero los cambios finales
>git merge

Camparar dos tag de mismo archivo
>git diff 5f30833f5c1a73996bb29dcf7664de4cb89a891a 9dd0f385743f1f2d428d8f0a0b7dc5f8cbf392a4

Permite volver a un commit en especifico
>git reset 5f30833f5c1a73996bb29dcf7664de4cb89a891a --hard


Primete traer una version anterior de uan archivo 
>git checkout 5f30833f5c1a73996bb29dcf7664de4cb89a891a archivo_nuevo.txt

Vamos crear un origen remoto de nuestros archivos
>git remote add origin http://192.168.50.127:5000/dmo/taller_01.git

Setear la url remoto a la que se quiere apuntar
>git remote set-url origin https://github.com/USERNAME/REPOSITORY.git

Indica  a que ruta remota estoy apuntando
>git remote -v

Traerme la ultima verison de directorio remoto
>git pull origin master

Subir los cambios al repositorio remoto
>git push origin master
